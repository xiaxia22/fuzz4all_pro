## 这一页要强调的方法

老师如果问“你具体怎么实现的”，你就按这 4 点答：

### 1. 输入重构方法
- 用 API 文档替换原先的语言特性提示源
- 每个 target 通过 `path_documentation` 指向对应 API 文档

### 2. 分类实现方法
- 通过离线脚本对文档内容、包名、类名、关键词和风险点做综合判定
- 输出分类结果到 `outputs/api_classification`

### 3. 运行时接入方法
- `profile_loader.py` 读取分类结果
- 给每个 API 生成：
  - `primary_tag`
  - `mutation_profile`
  - `priority_operators`

### 4. 闭环实现方法
- `target.py` 与 `JAVA.py` 负责生成、编译、记录反馈
- `runtime_rules_*.txt` 和 `runtime_prompt_*.txt` 负责把失败规则写回下一轮 prompt
